* Neovim Configuration

Minimal, efficient Neovim configuration following suckless philosophy, written in literate programming style.

** Philosophy

- *Minimal*: Only essential plugins and configurations
- *Efficient*: Lazy loading and fast startup
- *Lua-based*: Modern Lua configuration throughout
- *Documented*: Every setting explained

* Features

** Core Features
- Transparent background support
- Smart indentation and search settings
- Persistent undo with dedicated directory
- System clipboard integration

** Key Mappings
- Space as leader key
- Emacs-style navigation in insert mode
- Buffer and window management shortcuts
- Visual mode enhancements

** Language Support (via LSP)
- Lua (lua-language-server)
- Python (pyright)
- TypeScript/JavaScript (ts_ls)
- Ruby (ruby-lsp)
- Web technologies (HTML, CSS, JSON)
- Markdown (markdown-oxide, marksman)

** Plugin Ecosystem
- Telescope for fuzzy finding
- Treesitter for syntax highlighting
- nvim-cmp for completion
- Mason for LSP management
- Neo-tree for file exploration

* Usage

** Tangling Configuration

From Emacs:
1. Open any org file
2. Run =C-c C-v t= (or =M-x org-babel-tangle=)

Configuration files are written to =~/.config/nvim/=

** Directory Structure

#+begin_example
~/.config/nvim/
├── init.lua                 # Main configuration
├── lua/
│   └── pengling/           # Custom namespace
│       ├── plugins/        # Plugin configs
│       └── utils/          # Utility functions
└── ftplugin/              # Filetype settings
#+end_example
